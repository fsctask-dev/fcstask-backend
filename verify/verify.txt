при регистрации на курс у юзера есть почта в его профиле.

кнопка регистрации на курс -> Автоматическая проверка -> Ручная модерация админами при необходимости -> доступ к курсу

статусы пользователя:
unverified - у только что зарегистрировавшихся, нет доступа к элементам курса
verified - после верификации даем доступ
pending - в списке имейлов не был найден, отправлен на ручную проверку
rejected - отклонен админом


[client]->[API Handler]->[VerificationService]->[]
							                    |
							                    |WhiteList
							                    |Logger


хендлер принимает запрос валидирует данные вызывает методы сервиса возвращает ответ, логгируем автоматическое подтверждение
сервис принимает запрос проверяет наличие имейл в whitelist, проставляет verified или pending

WhiteList - список студентов курса(и соответственно их почт) из УО(я пологаю он будет доступен???) -- задается админами при создании курса

если админ одобряет pending юзера проставляем verified и логгируем действие отмечая ID админа проставившего статус
если админ реджектит простовляем соответствующий статус и логгируем

ручная проверка подучается будет для тех кто будет регистрироваться не по корпоротивной почте

допустимые переходы
unverified -> verified
unverified -> pending
pending -> verified
pending -> rejected
rejected -> penfing с флагом повторная
остальные переходы не допустимы
