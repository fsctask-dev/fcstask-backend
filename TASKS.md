# Задачи

* задачи поставлены до окончания срока окончания оценивания проекта

при возникновении сложности необходимо обращаться в первую очередь в чат, во вторую мне (@turbomuza)

Как делать задачу:
1. Чекаутимся в ветку main (`git checkout main`)
2. Пуллим изменения (`git pull origin main`)
3. Прогоняем тесты (cм. Makefile)
4. Создаем новую ветку (`git checkout -b feat/{название_задачи_коротко}`)
5. Делаем задачу
6. Комитим и создаем пулл-реквест


**Задачи вместе с идеей реализацией и проработка основных идей** 

1. Идея обсуждается с членами команды, записывается архитектура и возможные подводные камни
2. Как только получается модель, комитить ее в текстовом редакторе и шарить в чат
3. Как только пошарили пинговать @turbomuza для обсуждения выбранной архитектуры, проводится созвон
4. На проработку идеи не должно уходить **больше 2 дней**


Критерии:
* На каждой недели необходимо созваниваться с участниками командами (без меня, внутри, обсуждать идеи, прорабатывать процесс выполнения задачи и это документировать)
* В случае проблем -- **пинговать в чате @turbomuza**
* Необходимо чтобы был хотя бы 1 коммит в неделю, который должен быть > 100 cтрок с содержательными вещами по выполнению задач


**На каждую неделю необходимо иметь отчет о проделанной работе.**


**В случае не исполнению критерий оценка итоговая может быть сильно снижена, необходимое полное **командной** включение в проект**
